Fictional Character API Documentation

Host: localhost:8080

Resource Directory: /fictionalCharacterAPI_war

Resource:

  • /service


/character/id

Method: GET

Required Parameters

  • 1. Id

    • Optional Parameters

  • None

    • Example: http://3.142.182.83:8080/fictionalCharacterAPI_war/service/character/id/1

    The /id resource uses the GET method and a number parameter to fetch a character in the fictional character database according to its id

  • 2. Name

    • Example: http://3.142.182.83:8080/fictionalCharacterAPI_war/service/character/name/Luke%20Skywalker

    The /name resource uses the GET method and a name parameter to fetch a character in the fictional character database according to its name

  • 3. Media

    • Example: http://3.142.182.83:8080/fictionalCharacterAPI_war/service/character/media/book

    The /name resource uses the GET method and a media parameter to fetch a character in the fictional character database according to its media release (book, movie, series, etc)

  • 4. Franchise

    • Example: http://3.142.182.83:8080/fictionalCharacterAPI_war/service/character/franchise/Lord%20of%20the%20Rings

    The /name resource uses the GET method and a franchise parameter to fetch a character in the fictional character database according to the franchise the character belongs to

  • 5. Species

    • Example: http://3.142.182.83:8080/fictionalCharacterAPI_war/service/character/species/human

    The /name resource uses the GET method and a species parameter to fetch a character in the fictional character database according to its species

  • 6. Alignment

    • Example: http://3.142.182.83:8080/fictionalCharacterAPI_war/service/character/alignment/good

    The /name resource uses the GET method and an alignment parameter to fetch a character in the fictional character database according to its alignment (good, evil, chaotic neutral, etc)

  • 7. Multi-Parameter Request

    • Example: http://3.142.182.83:8080/fictionalCharacterAPI_war/service/character/media/book/name/Gandalf

    The Multi-parameter request is made by using 2 of the resources from 2-6. The /character resource supports one resource like media with a search parameter, and can then search the results with another resource like searching book characters for good ones.

    Method: POST

    Required Parameters

  • 1. name
  • 2. media
  • 3. franchise
  • 4. species
  • 5. alignment

    • Optional Parameters

  • None

    • Example: http://3.142.182.83:8080/fictionCharacterAPI_war/service/insert

    Send json data defining the above properties to the endpoint to insert a new fictional character into the database
